## ğŸ“‹ Issue Guidelines

### Good Issue Reports Include:

- **Clear title** that summarizes the issue
- **Detailed description** of the problem or request
- **Context** about your use case
- **Screenshots or examples** when applicable
- **Respectful tone** and constructive feedback

### Please Avoid:

- Duplicate issues (search first!)
- Vague descriptions without details
- Multiple unrelated topics in one issue
- Demanding or rude language

## âš¡ Issue Triage Process

Issues go through the following process:

1. **Triage**: New issues are labeled and reviewed by maintainers
2. **Investigation**: Issues are analyzed and prioritized
3. **Planning**: Approved issues are added to development roadmap
4. **Implementation**: Features/fixes are implemented in the private repository
5. **Closure**: Issues are closed when resolved in a release

## ğŸ·ï¸ Labels

We use labels to organize issues:

- `bug`: Something isn't working
- `enhancement`: New feature or request
- `question`: Question or support request
- `triage`: Needs review from maintainers
- `duplicate`: Duplicate of existing issue
- `wontfix`: Will not be addressed
- `in-progress`: Currently being worked on
- `help wanted`: Community help welcome
